## El lenguaje del motor: PL/SQL
SQL estándar es declarativo (tú dices _qué_ quieres, pero no _cómo_ hacerlo). PL/SQL es la extensión de Oracle que permite añadir lógica de programación:
- **Variables y constantes.**
- **Estructuras de control:** `IF-THEN-ELSE`, bucles `FOR` y `WHILE`.

## Caracteristicas
Tiene algoritmos bastantes avanzados para gestionar datos que son mas tecnicos pero puedes ayudar:
**Concurrency Control (ACID):** Oracle es famoso por su manejo de transacciones. Si 1,000 personas intentan comprar el mismo boleto de avión al mismo tiempo, Oracle garantiza que solo una lo logre
**Optimizer:** Tiene un "cerebro" interno que analiza tu consulta SQL y decide cuál es el camino matemático más corto para encontrar los datos

## Ventajas de Oracle Database
Oracle brilla en entornos donde el fallo no es una opción:
- **Rendimiento en Cargas Masivas:** Mientras que MySQL puede sufrir con tablas de billones de registros, Oracle utiliza técnicas como el _Partitioning_ y el _Parallel Execution_ para procesar datos a una velocidad que pocos alcanzan.
    
- **Disponibilidad Extrema (RAC):** Con **Oracle Real Application Clusters**, puedes tener varios servidores accediendo a la misma base de datos. Si un servidor físico explota, la base de datos ni siquiera se entera; sigue funcionando sin perder un solo segundo.
    
- **Base de Datos Convergente:** A diferencia de otros que te obligan a usar bases de datos distintas (una para JSON, otra para tablas, otra para mapas), Oracle maneja **todo** en un solo motor. Esto simplifica tu trabajo si mezclas diferentes tipos de datos.
    
- **Automatización Total:** Gracias a su versión _Autonomous_, la base de datos se pone parches de seguridad,

## Desventajas de Oracle Database
- **El Costo (Licenciamiento):** Es, por mucho, la opción más cara
- **Complejidad de Administración:** No es "instalar y listo". Requiere de un **DBA (Database Administrator)** experto. Un Oracle mal configurado puede ser más lento que un MySQL básico.
- **Curva de Aprendizaje:** dominar las interioridades de Oracle y su lenguaje **PL/SQL** toma mucho más tiempo que aprender la sintaxis de SQL Server o MySQL.
- **Bloqueo de Proveedor (Vendor Lock-in):** Una vez que tu empresa construye todo sobre las funciones específicas de Oracle, es extremadamente difícil y costoso mudarse a otro competidor.
